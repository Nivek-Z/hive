# Hive（蜂巢）实时聊天社区 — 实现计划

> **For agentic workers:** 按里程碑顺序在本会话内逐任务执行（inline execution）。每完成一个里程碑：编译 → 测试 → git commit。本文件是跨会话的唯一事实来源（context 压缩后以此恢复状态）。

**Goal:** 一个 Discord 风格的 Web 实时聊天社区（Java 课程大作业），支持群聊/私聊/树形频道（群中群）/角色权限/成就彩蛋，单 jar 部署，自带便携 MySQL，开箱即用。

**Architecture:** Spring Boot 单体应用：REST API（账号/社区/管理） + 原生 WebSocket 自定义 JSON 协议（实时消息），MyBatis 手写 SQL 操作 MySQL 8；前端为纯原生 HTML/CSS/JS 单页应用，打包进 jar 的 static 目录。成就系统用 Spring 事件（观察者模式）驱动。为后续 Android 客户端预留：所有能力均走 REST+WS，无服务端渲染。

**Tech Stack:** Java 25（编译 target 21）· Spring Boot 3.5.x · MyBatis 3 · MySQL 8.0.28（便携版，tools/ 内）· spring-security-crypto（BCrypt）· 手写 JWT（HS256，无第三方依赖，答辩亮点）· 原生 JS 前端 · JUnit 5

## 2. 数据库设计（14 张表，MySQL 8 / utf8mb4）

> 完整 DDL 写入 `hive/src/main/resources/db/schema.sql`，要点：树形结构（channels.parent_id 自引用=群中群）、位掩码权限（roles.permissions BIGINT）、多对多（member_roles/reactions/user_achievements）、软删除（messages.deleted）、中文全文索引（ngram）、消息毫秒时间戳、全部外键 + 级联删除。

```sql
users(id PK, username UNIQUE, password_hash BCrypt, nickname, avatar_color, avatar_url,
      bio, created_at, last_seen_at)
hives(id PK, name, description, icon_color, owner_id→users, created_at)
invites(id PK, code CHAR(8) UNIQUE, hive_id→hives 级联, creator_id, max_uses(0=∞),
        used_count, expires_at NULL=永久, created_at)
hive_members(id PK, hive_id, user_id, hive_nickname, muted_until, joined_at,
             UNIQUE(hive_id,user_id))
channels(id PK, hive_id NULL(DM时), parent_id→channels 自引用【群中群】,
         type ENUM('CATEGORY','TEXT','DM'), name, topic, position, created_at)
channel_members(id PK, channel_id, user_id, UNIQUE(channel_id,user_id))  -- 仅 DM 用
messages(id PK, channel_id, sender_id NULL(系统消息), type ENUM('TEXT','IMAGE','SYSTEM'),
         content TEXT, reply_to_id, deleted, created_at DATETIME(3),
         INDEX(channel_id,id), FULLTEXT(content) WITH PARSER ngram)
reactions(id PK, message_id, user_id, emoji, UNIQUE(message_id,user_id,emoji))
friendships(id PK, requester_id, addressee_id, status ENUM('PENDING','ACCEPTED'),
            created_at, UNIQUE(requester_id,addressee_id))
roles(id PK, hive_id, name, color, permissions BIGINT 位掩码, position, is_default)
member_roles(id PK, hive_id, user_id, role_id, UNIQUE(user_id,role_id))
read_states(id PK, user_id, channel_id, last_read_message_id, updated_at,
            UNIQUE(user_id,channel_id))
achievements(id PK, code UNIQUE, name, description, emoji, secret, points)
user_achievements(id PK, user_id, achievement_id, unlocked_at, UNIQUE(user_id,achievement_id))
files(id PK, uploader_id, stored_name UNIQUE, original_name, mime, size_bytes, created_at)
```

事务点（答辩讲解用）：加入邀请（校验+used_count自增+插成员+发系统消息）、踢人（删成员+删角色+系统消息）、删蜂巢（级联）。

## 3. 权限位设计（Discord 同款思路，答辩亮点）

```java
public final class Permissions {
    public static final long ADMINISTRATOR    = 1L << 0;   // 等同所有权限
    public static final long MANAGE_HIVE      = 1L << 1;   // 改蜂巢资料
    public static final long MANAGE_CHANNELS  = 1L << 2;   // 频道增删改
    public static final long MANAGE_ROLES     = 1L << 3;   // 角色管理/分配
    public static final long KICK_MEMBERS     = 1L << 4;
    public static final long MUTE_MEMBERS     = 1L << 5;
    public static final long DELETE_MESSAGES  = 1L << 6;   // 删他人消息
    public static final long CREATE_INVITE    = 1L << 7;
    public static final long MENTION_EVERYONE = 1L << 8;
    public static final long SEND_MESSAGES    = 1L << 9;
    public static final long ATTACH_FILES     = 1L << 10;
    public static final long ADD_REACTIONS    = 1L << 11;
    public static final long DEFAULT_MEMBER   = CREATE_INVITE|SEND_MESSAGES|ATTACH_FILES|ADD_REACTIONS;
    public static boolean has(long perms, long bit) {
        return (perms & ADMINISTRATOR) != 0 || (perms & bit) != 0;
    }
}
```

生效权限 = 该成员所有角色 permissions 按位 OR ∪ 默认角色"工蜂"(is_default=1, DEFAULT_MEMBER)；蜂巢 owner 直接全权；禁言(muted_until>now) 覆盖 SEND_MESSAGES/ADD_REACTIONS。建巢时自动建两个角色：工蜂(默认)、管理员(预设管理位)。

## 4. REST API 一览（前缀 /api，除 auth 外都需 `Authorization: Bearer <JWT>`）

```
认证   POST /auth/register {username,password,nickname}   POST /auth/login → {token,user}
用户   GET/PUT /users/me · PUT /users/me/password · GET /users/{id}
       GET /users/me/heatmap(过去365天逐日消息数) · GET /users/me/achievements
蜂巢   POST/GET /hives · GET/PUT/DELETE /hives/{id} · POST /hives/{id}/leave
       GET /hives/{id}/members · DELETE /hives/{id}/members/{uid}(踢)
       POST/DELETE /hives/{id}/members/{uid}/mute {minutes}
       POST /hives/{id}/invites {maxUses,expiresHours} · POST /invites/{code}/join
       GET /hives/{id}/stats(7日活跃曲线/发言排行)
频道   POST /hives/{id}/channels {name,type,parentId} · PUT/DELETE /channels/{id}
       GET /channels/{id}/messages?before&limit=50 · POST /channels/{id}/read {lastMessageId}
消息   DELETE /messages/{id} · POST /messages/{id}/reactions {emoji} · DELETE .../reactions/{emoji}
好友   GET /friends · POST /friends/requests {username} · GET /friends/requests
       POST /friends/requests/{id}/accept · DELETE /friends/requests/{id} · DELETE /friends/{uid}
私聊   POST /dms/{userId} → {channelId} · GET /dms(会话列表带最后一条消息)
角色   GET/POST /hives/{id}/roles · PUT/DELETE /roles/{id}
       PUT /hives/{id}/members/{uid}/roles {roleIds}
文件   POST /files (multipart) → {url:"/uploads/xxx"}
搜索   GET /search/messages?q&hiveId (MySQL ngram 全文检索)
彩蛋   POST /eggs/konami (解锁隐藏成就)
```

统一响应 `{code:0,msg:"ok",data:...}`；业务错误抛 BizException(code,msg)，GlobalExceptionHandler 统一兜底。

## 5. WebSocket 协议（`/ws?token=JWT`，JSON 信封 `{type,data}`）

```
C→S  MSG_SEND {channelId,content,type,replyToId,nonce} · TYPING {channelId} · PING
S→C  READY {user,onlineUserIds}                    连接成功
     MSG_NEW {message,nonce}                       新消息(含发送者快照)→推给频道可见者
     MSG_DELETED {channelId,messageId}
     REACTION_UPDATE {channelId,messageId,reactions[]}
     TYPING {channelId,userId,nickname}            3秒节流
     PRESENCE {userId,online,lastSeenAt}           上下线广播给相关用户
     HIVE_EVENT {kind,hiveId,payload}              MEMBER_JOINED/KICKED/CHANNEL_CREATED/ROLE_CHANGED...→刷新侧栏
     ACHIEVEMENT_UNLOCKED {achievement}            触发前端金色弹窗+音效
     EGG {effect,payload}                          confetti/bee_rain 全频道特效广播
     ERROR {code,message} · PONG
```

推送范围：频道消息 → 该蜂巢所有在线成员（客户端按当前频道过滤+未读计数）；DM → 两个参与者。`WsSessionRegistry`: `ConcurrentHashMap<Long userId, Set<WebSocketSession>>`（多端登录）。

斜杠命令（服务端处理，结果持久化为 SYSTEM 消息）：`/roll [上限]`、`/rps 石头|剪刀|布`、`/8ball 问题`、`/fortune`（今日运势，userId+日期作种子→每天固定）、`/help`。关键词彩蛋：内容含"生日快乐"或🎉 → EGG confetti。前端 Konami 秘技(↑↑↓↓←→←→BA) → 蜜蜂雨+隐藏成就。

## 6. 成就系统（观察者模式，答辩亮点）
