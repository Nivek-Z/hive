# Hive TUI UI 与菜单优化设计

## 目标

把 Hive TUI 从“可用的聊天终端”提升到更像完整聊天客户端的初版界面：主消息流仍然突出，左侧先展示群聊再展示频道，右侧展示在线与服务器摘要，`Tab` 打开随焦点变化的上下文菜单。登录页也要与聊天页保持同一套终端面板风格，并在菜单中提供注册入口。

## 范围

本轮实现包含：

- 聊天页三栏布局：左侧群聊和频道树，中间消息流，右侧在线/当前群聊/服务器摘要。
- 左侧栏顺序调整为先 `hives`，再 `channels`。
- 右侧在线区域使用绿色圆点显示在线状态；没有真实成员接口时只展示当前用户或占位说明，不伪造好友/成员列表。
- `Tab` 打开上下文菜单，`Esc` 关闭菜单。
- 菜单打开时，`Up`/`Down` 移动高亮项，`Enter` 执行当前项。
- 菜单内容根据当前焦点变化：
  - 登录页：登录、注册、服务器设置、退出。
  - 频道导航：打开/收放、切换群聊、刷新频道、设置。
  - 消息阅读：跳到最新、成员列表、好友、设置。
  - 输入区：发送消息、好友、在线成员、设置。
- 登录页改成居中的带边框终端面板，底部提示 `Tab menu`、`Enter login`、`Ctrl+C quit`。

本轮不实现：

- 真实注册接口调用。注册菜单项只提供入口和“接口未接入”反馈。
- 真实好友列表接口调用。
- 真实在线成员详情接口调用。当前 WebSocket 只有在线 ID 时，可展示计数或占位，不猜测昵称。
- 配置编辑保存。设置面板仍显示当前配置和“远程设置接口未接入”。

## 交互

`Tab` 是统一菜单入口。菜单打开后，普通焦点操作暂停，按键解释如下：

- `Up`/`Down`：移动菜单高亮。
- `Enter`：执行高亮菜单项。
- `Esc`：关闭菜单。
- `Ctrl+C`：退出程序。

菜单关闭时，保留现有操作：

- `Left`/`Right`：切换导航、消息、输入焦点。
- `Up`/`Down`：导航焦点移动频道树；消息或输入焦点滚动消息。
- `Enter`：导航焦点打开频道或收放分类；输入焦点发送消息。
- 普通字符输入：自动切到输入区。


窄屏时隐藏右侧栏，状态栏仍保留 `Tab menu` 提示，菜单内容显示在主区域。

菜单使用克制的边框和高亮行，不用大块装饰：

```text
COMPOSER MENU
> 发送消息        Enter
  好友            F
  在线成员        M
  设置            ,
```

登录页使用同一套边框风格：

```text
+--------------------------------+

| > Username: nivek              |
|   Password: ********           |
|                                |
| Tab menu | Enter login         |
| server chhat.nievkz.org        |
+--------------------------------+
```

## 代码边界

短期继续在 `tui/internal/app` 内实现，避免在初版 UI 还快速变化时拆太多包。为了控制复杂度，新增逻辑按小函数组织：

- 群聊/频道渲染：`navLines` 继续负责左侧栏，增加 hives 区块。
- 布局：`chatView` 根据窗口宽度选择两栏或三栏。
- 右侧栏：新增右侧信息函数，显示在线、当前群聊和服务器信息。
- 菜单：新增菜单状态、菜单项生成、菜单选择执行函数。
- 登录页：重写 `loginView` 为带边框面板，并复用菜单渲染。

## 测试策略

新增 Go 单元测试覆盖：

- 左侧栏先显示群聊名，再显示频道名。
- 宽屏聊天页显示右侧在线栏和绿色在线圆点。
- `Tab` 打开菜单，菜单随焦点显示不同项。
- 菜单打开时 `Up`/`Down` 移动高亮，`Enter` 执行选择。
- `Esc` 能关闭菜单。
- 登录页显示边框面板，登录菜单包含注册入口。
- 视图仍受窗口宽高约束。
