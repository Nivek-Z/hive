# Hive TUI 初版打磨设计

## 目标

把当前 Go TUI 从“能用的双栏工具界面”调整为“主体突出、简约、终端感强”的聊天界面。初版仍只完成稳定聊天主流程，同时给后续配置菜单、好友、在线群员等功能预留入口和代码边界，不伪造后端尚未提供的能力。

## 范围

本次实现包含：

- 主消息区作为视觉主体，左侧频道树弱化。
- 顶部显示当前频道名和 topic。
- 消息按参考图风格渲染：作者名醒目、时间弱化、内容缩进显示。
- 左侧频道树修复首个频道显示问题。
- 支持嵌套频道树展开和折叠。
- `Enter` 在分类行上切换展开/折叠，在文字频道行上切换频道并加载历史。
- 切换频道后拉取最近 50 条消息，并用最后一条消息标记已读。
- 状态栏显示短反馈，如 `opened #大厅`、`collapsed 常规`、`sent`。
- 预留配置、好友、群员入口；如果接口未接入，显示占位面板，不使用假数据。

本次不实现：

- 好友列表真实数据。
- 在线群员真实数据。
- 配置菜单保存。
- 私聊、文件上传、表情反应、回复引用、成员管理等扩展功能。

## 交互

主模式仍保留三种焦点：导航、消息、输入。

- `Left` / `Right`：在导航、消息、输入之间切换。
- `Up` / `Down`：导航焦点时移动频道树光标；消息或输入焦点时滚动消息。
- `Enter`：导航焦点下，分类行展开或折叠，文字频道行打开频道；输入焦点下发送消息。
- 普通字符输入：自动切到输入焦点。
- `F`：打开好友占位面板。
- `M`：打开群员占位面板。
- `,`：打开配置占位面板。
- `Esc`：关闭占位面板或回到导航焦点。

## 视觉

布局使用终端稳定字符，避免 Windows 终端编码问题：

```text
#大厅  什么都能聊的地方
------------------------------------------------------------

nivek   06-14 14:14
  /uploads/9a1ef37a68ed4a44dbbdec3b91fbbb40d.jpg

zkw     06-14 15:01
  你是不是反革命？

------------------------------------------------------------
> fgm干嘛呢
connected | INPUT | F friends | M members | , config | sent
```

导航焦点时左侧频道树占用固定窄列；非导航焦点时仍显示但不抢主消息区。树符号使用 ASCII：

- `> `：导航光标所在行。
- `* `：当前打开的文字频道。
- `- `：已展开分类。
- `+ `：已折叠分类。
- `# `：文字频道。
- `[n]`：未读数。

## 代码边界

短期仍在 `internal/app` 内实现，但把逻辑拆成小函数：

- 可见频道树：根据折叠状态生成导航行。
- 导航操作：移动光标、展开折叠、打开频道。
- 消息格式：格式化时间、作者、内容缩进和换行。
- 面板模式：只存储当前占位面板类型，不访问未实现接口。

后续如果继续增加好友和成员真实接口，再把这些函数迁移到 `internal/nav`、`internal/chatview`、`internal/panels` 等包。

## 测试策略

新增 Go 单元测试覆盖：

- 左侧第一个频道名称必须显示。
- 分类 `Enter` 后能折叠，再次 `Enter` 后展开。
- 折叠分类时隐藏子频道。
- 文字频道 `Enter` 会调用 API 加载历史并更新当前频道。
- 消息显示时间，内容使用缩进两行结构。
- `F`、`M`、`,` 打开占位面板，`Esc` 关闭。
- 视图宽高仍受窗口约束。
